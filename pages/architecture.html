6. Ключевые компоненты и их функции

Детальное описание основных компонентов системы и их ответственности.

🎨 Frontend Component

Ответственность: отображение UI, взаимодействие с пользователем, клиентская валидация, локальное кеширование.

Технологии: HTML5, CSS3, JavaScript/React, LocalStorage API.

🔐 Authentication Service

Ответственность: валидация credentials, выдача JWT/tokens, проверка ролей, управление сессиями.

Технологии: JWT, OAuth2, bcrypt/argon2, Redis для token blacklist.

⚙️ API Gateway / Router

Ответственность: маршрутизация запросов, middleware (логирование, rate-limiting, CORS), валидация данных.

Технологии: Express/Fastify, OpenAPI spec, middleware stack.

📊 Business Logic Layer

Ответственность: обработка данных, бизнес-правила, расчеты, генерация отчетов.

Технологии: TypeScript, Design patterns (service, repository), unit-тесты.

🗄️ Data Access Layer

Ответственность: работа с БД, ORM/query builder, миграции, транзакции.

Технологии: PostgreSQL, Sequelize/TypeORM, Migrations, Connection pooling.

💬 WebSocket Service

Ответственность: real-time сообщения, двусторонняя коммуникация, управление комнатами чата.

Технологии: Socket.IO, Redis Adapter для масштабирования, Message Queue.

⚡ Cache Layer

Ответственность: кеширование данных, сессии, rate-limiting, очередь задач.

Технологии: Redis, TTL policies, Pub/Sub для broadcast.

☁️ File Storage

Ответственность: загрузка файлов, хранение, CDN доставка, управление метаданными.

Технологии: AWS S3 / MinIO, Multer, signed URLs для доступа.

📧 Notification Service

Ответственность: отправка email, SMS, push-уведомлений, templating.

Технологии: SendGrid, Twilio, Firebase Cloud Messaging, nodemailer.

📈 Monitoring & Logging

Ответственность: логирование событий, метрики производительности, алерты, трассировка.

Технологии: ELK Stack, Prometheus, Grafana, Datadog, Winston.

7. Основные flow-диаграммы (сценарии)

Последовательные диаграммы основных процессов системы.

🔐 Flow: Аутентификация пользователя

sequenceDiagram participant User participant Frontend participant API participant Auth participant DB User->>Frontend: 1. Вводит login/пароль Frontend->>API: 2. POST /auth/login API->>Auth: 3. Валидация credentials Auth->>DB: 4. Поиск пользователя DB-->>Auth: 5. Проверка хеша пароля Auth-->>API: 6. Выдача JWT token API-->>Frontend: 7. Возврат token Frontend->>Frontend: 8. Сохранение в localStorage Frontend-->>User: 9. Редирект в личный кабинет

💬 Flow: Сообщение в чат (User → Bot → Operator)

sequenceDiagram participant User participant Frontend participant WebSocket participant Bot participant Queue participant Operator User->>Frontend: 1. Пишет сообщение Frontend->>WebSocket: 2. Emit message WebSocket->>Bot: 3. Передача сообщения боту Bot->>Bot: 4. Анализ и поиск ответа Bot-->>WebSocket: 5. Ответ (если есть в БД) WebSocket-->>Frontend: 6. Получение ответа Frontend-->>User: 7. Показ ответа Note over User: Если бот не смог решить... Bot->>Queue: 8. Добавление в очередь Queue->>Operator: 9. Уведомление оператора Operator->>WebSocket: 10. Оператор подключается WebSocket->>Frontend: 11. Оповещение: оператор присоединился Frontend-->>User: 12. Чат с оператором открыт

📝 Flow: Подача заявки (Application Submission)

sequenceDiagram participant Student participant Frontend participant API participant FileStorage participant DB participant Email Student->>Frontend: 1. Заполняет форму заявки Frontend->>API: 2. POST /applications (с файлами) API->>FileStorage: 3. Загрузка файлов в S3/MinIO FileStorage-->>API: 4. Возврат URLs файлов API->>DB: 5. Сохранение заявки (status: pending) DB-->>API: 6. ID заявки API->>Email: 7. Отправка уведомления Email-->>Student: 8. Email: Заявка принята API-->>Frontend: 9. Успех, ID заявки Frontend-->>Student: 10. Показ статуса заявки
